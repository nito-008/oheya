# Design Guidelines

## Layout Surfaces

- Avoid defaulting to card-based layouts for settings pages and other primary screens.
- Prefer open layouts with spacing, dividers, and typography before adding bordered or filled containers.

## Shared Actions

- Reusable compact actions should use the shared pill button under `src/components/ui/button`.
- Prefer icon + short message buttons for settings-style actions instead of icon-only controls.
